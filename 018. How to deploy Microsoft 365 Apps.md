# Lecture Notes: Deploying Microsoft 365 Apps to Windows 10 Using Intune

## Overview

* **Topic:** Deploying **Microsoft 365 Apps for Windows 10** using Microsoft Intune
* **Use Case:** Centrally install Microsoft 365 applications on Intune-managed Windows devices
* **Result:** Microsoft 365 Apps are deployed, managed, and updated through Intune

---

## Background Information

* Before deploying applications, they must be **added to Intune**
* One supported app type is:

  * **Microsoft 365 Apps for Windows 10 or later**
* This app type allows deployment of:

  * Microsoft 365 Apps (formerly Office 365 ProPlus)
  * Optional licensed apps:

    * Microsoft Project Online (Desktop)
    * Microsoft Visio Online Plan 2

> ⚠️ **Office 365 ProPlus** has been renamed to **Microsoft 365 Apps**

---

## Scenario Covered

* Target OS: **Windows 10**
* Device state:

  * No previous Office version installed
* Deployment method:

  * Intune app deployment using **Configuration Designer**

---

## Admin Portal Used

* **Portal:** Microsoft Endpoint Manager
* **URL:** `https://endpoint.microsoft.com`
* Required role:

  * Global Administrator **or**
  * Intune Service Administrator

---

## Step 1: Add Microsoft 365 Apps in Intune

### Navigation

1. Sign in to **Microsoft Endpoint Manager Admin Center**
2. Click **Apps**
3. Under **By platform**, select **Windows**
4. Click **Add**

---

### Select App Type

1. From **App type**, choose:

   * **Microsoft 365 Apps for Windows 10 or later**
2. Click **Select**

---

## Step 2: App Information

* App details are auto-generated
* You can customize:

  * App name
  * Description

### Example

* **Name:** MSF Surat – Microsoft 365 Apps for Windows 10
* Other fields: Default

Click **Next**

---

## Step 3: Configure Microsoft 365 Apps (Configuration Designer)

### Configuration Method

Two options are available:

* **Configuration Designer** (recommended)
* Enter **XML configuration manually**

➡ This walkthrough uses **Configuration Designer**

---

### Select Office Apps

* Choose required Microsoft 365 apps:

  * Word
  * Excel
  * PowerPoint
  * Outlook
  * OneNote
  * Teams
* **Skype for Business** is excluded (Teams is the replacement)

> **Best practice:** Bundle required apps into a **single Intune app**

---

### Additional Office Apps (Optional)

* Microsoft Project Online Desktop
* Microsoft Visio Online Plan 2

> Select these **only if licenses are available**
> (Not selected in this example)

---

### Platform and Format Settings

* **Architecture:** 64-bit
* **Default file format:** Office Open XML

---

### Update Channel

* **Recommended:** Monthly Enterprise Channel

  * Predictable monthly updates
  * Regular feature delivery

---

### Remove Existing Office Versions

* Enable removal of **pre-existing MSI-based Office apps**
* Required to avoid installation failures

---

### Version Selection

* Options:

  * Install **latest version** (selected)
  * Install a **specific version**

---

### Shared Computer Activation

* Enable if:

  * Multiple users share the same device

---

### License Acceptance

* **Accept Microsoft license terms automatically**
* End users are not prompted

---

### Microsoft Search in Bing

* Select **No** (as per example)

---

### Language Settings

* Default: OS language
* Additional languages can be added if required

Click **Next**

---

## Step 4: Assignments

### Assignment Types

* **Required**

  * App installs automatically
* **Available for enrolled devices**

  * App appears in Company Portal for manual installation
* **Uninstall**

  * Removes app from devices

> In production, **Required** is most common
> This demo uses **Available** for visibility

---

### Assignment Configuration

1. Select **Available for enrolled devices**
2. Click **Add group**
3. Choose **Test Users** group
4. Click **Select**
5. Click **Next**

---

## Step 5: Review and Create

1. Review configuration summary
2. Click **Create**
3. Confirmation message displayed:

   * *Microsoft 365 Apps for Windows 10 created successfully*

---

## Step 6: Manual Installation from Company Portal (Client Side)

### User Scenario

* User signed in:

  * Member of **Test Users** group
* Assignment type:

  * **Available**

---

### Install from Company Portal

1. Open **Company Portal** app
2. Go to **Apps**
3. Perform **Sync**:

   * Settings → Sync
4. Wait ~10 minutes
5. Microsoft 365 Apps appear
6. Click **Install**

---

### Installation Progress

* Apps are downloaded and installed automatically
* Installation may take several minutes

---

## Step 7: Verification

### Application Verification

* Start Menu shows:

  * Word
  * Excel
  * PowerPoint
  * Outlook

---

### Functional Test

1. Open **Microsoft Word**
2. Create a blank document
3. Confirm successful launch

---

### Installed Apps Check

1. Open **Run**
2. Type `appwiz.cpl`
3. Confirm:

   * **Microsoft 365 Apps for Enterprise** installed

---

### Company Portal Status

* Company Portal → **Downloads & updates**
* Status shows:

  * **Installed successfully**

---

## Key Takeaways

* Intune provides native deployment of Microsoft 365 Apps
* Configuration Designer simplifies setup
* Supports:

  * App bundling
  * Update channel control
  * License handling
* Assignment flexibility supports:

  * Automatic deployment
  * User-driven installs
* Ideal for enterprise-scale Office deployments

---

## Summary

* Microsoft 365 Apps were deployed using Intune
* App appeared as a single entry in Intune
* Installation was user-initiated via Company Portal
* Deployment completed successfully and verified
* Recommended approach for managing Office apps on Windows devices
