# Lecture Notes: Adding a New Azure Active Directory Tenant

## Overview

* **Topic:** Creating a **new Azure Active Directory (Azure AD) tenant** using the Azure AD Admin Center
* **Use Case:** Replacing an expired trial tenant in a training or test environment
* **Result:** A new Azure AD tenant with an active **Enterprise Mobility + Security (EMS) E5 trial** subscription

---

## Background Context

* The existing tenant (`amazon.onmicrosoft.com`) was using a **trial subscription**
* The trial expired, making it unsuitable for continued lab or training use
* A **new Azure AD tenant** is required with a **fresh trial subscription**
* This method uses an **existing Global Administrator account**

> A separate video already covered tenant creation using the Azure portal.
> This session focuses on creating a tenant using the **Azure AD Admin Center**.

---

## Prerequisites

* An account with **Global Administrator** permissions
* Access to the **Azure AD Admin Center**
* Internet connectivity

---

## Admin Portal Used

* **Portal:** Azure Active Directory Admin Center
* **URL:** `https://aad.portal.azure.com`

---

## Step 1: Access Azure AD Admin Center

1. Sign in to **Azure AD Admin Center**
2. Authenticate using a **Global Administrator** account
3. On the home page, click **Azure Active Directory**

---

## Step 2: View Existing Tenants

* Current tenant visible:

  * **Primary domain:** `amazon.onmicrosoft.com`
* From the top menu, click **Manage tenants**
* Existing tenant details are displayed

---

## Step 3: Create a New Azure AD Tenant

### Start Tenant Creation

1. Click **Create**
2. On the **Basics** tab:

   * Select **Azure Active Directory**
3. Click **Next: Configuration**

---

### Configuration Details

Provide the following details:

* **Organization name:** `Amazon 2.0` (example)
* **Initial domain name:** `msfsuratv2`
* **Country or region:** United States

  * Recommended for testing if certain services are country-restricted

Click **Next: Review + Create**

---

### Validation and Creation

1. Verify validation status shows **Passed**
2. Review entered information
3. Click **Create**
4. Complete CAPTCHA verification
5. Submit request

---

## Step 4: Confirm Tenant Creation

* Message displayed: **Tenant creation is in progress**
* After completion:

  * New tenant created successfully
  * **Primary domain:** `msfsuratv2.onmicrosoft.com`
* Click the link to navigate to the **new tenant**

---

## Step 5: Verify Tenant Details

### Tenant Overview

* **Organization name:** Amazon 2.0
* **Primary domain:** `msfsuratv2.onmicrosoft.com`
* **License type:** Azure Active Directory Free

---

### Tenant Management View

1. Click **Manage tenants**
2. Confirm:

   * Two tenants exist:

     * Old tenant
     * New tenant (Amazon 2.0)
3. New tenant is set as the **default tenant**

---

## Step 6: Verify Initial User and Role

* When a new tenant is created:

  * The creator becomes the **first user**
  * The user is automatically assigned the **Global Administrator** role
* Currently:

  * Only **one user** exists in the new tenant

---

## Step 7: Activate EMS E5 Trial Subscription

### Why EMS E5 Is Required

* Needed for:

  * Microsoft Intune
  * Device management
  * Identity and security labs

---

### Steps to Activate Trial

1. In the new tenant, go to **Licenses**
2. Under **Quick tasks**, click **Get a free trial**
3. Select **Enterprise Mobility + Security E5**
4. Click **Free trial**
5. Click **Activate**

---

### Confirmation

* Message displayed:

  * **Successfully activated Enterprise Mobility + Security E5 trial**

---

## Final Outcome

✔ New Azure AD tenant created
✔ Domain: `msfsuratv2.onmicrosoft.com`
✔ EMS E5 trial subscription active
✔ Tenant ready for Intune and identity labs

---

## Key Takeaways

* Azure AD Admin Center can be used to create new tenants
* Trial expiration requires tenant replacement in lab environments
* Tenant creator automatically becomes:

  * First user
  * Global Administrator
* EMS E5 trial enables full Intune and security feature testing

---

## Summary

* A new Azure AD tenant was created successfully
* EMS E5 trial was activated for training purposes
* The environment is now ready for:

  * Microsoft Intune configuration
  * Device enrollment
  * Identity and access management labs
