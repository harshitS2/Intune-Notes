# Lecture Notes: Configuring Company Portal Branding in Microsoft Intune

## Overview

* **Topic:** Configuring **Company Portal branding** in Microsoft Intune
* **Use Case:** Customize the Company Portal app and website to reflect organizational branding and support information
* **Result:** End users see a branded, professional portal experience when accessing company resources

---

## What Is Company Portal Branding?

* Company Portal is used by end users to:

  * Enroll devices
  * Install applications
  * Access company resources
  * Find IT support information
* Branding applies to:

  * Company Portal app (Windows, Android)
  * Company Portal website
  * Intune-managed app experience

Branding improves **user trust, usability, and corporate identity**.

---

## Administrative Capabilities

As an administrator, you can:

✔ Customize organization name and logo
✔ Configure theme and text colors
✔ Upload brand images
✔ Add IT support contact details
✔ Configure privacy information
✔ Control app source visibility
✔ Hide or show specific portal features

---

## Admin Portal Used

* **Portal:** Microsoft Endpoint Manager
* **URL:** `https://endpoint.microsoft.com`

---

## Navigation Path

1. Sign in to **Microsoft Endpoint Manager Admin Center**
2. Click **Tenant administration**
3. Under **End user experience**, select **Customization**

---

## Customization Policy Options

* **Default customization policy**

  * Applies to all users
* **Custom policies**

  * Can be created for group-based targeting

For simplicity, this session edits the **Default customization policy**.

---

## Step-by-Step: Configure Company Portal Branding

### 1. Edit Default Customization Policy

1. Click **Default customization policy**
2. Select **Edit**

---

### 2. Organization Details

* **Organization name:** `MSF Surat` (example)
* **Theme color:**

  * Choose from predefined colors
  * Or select **Custom** and enter a hex color code
* **Text color:** Black (example)

---

### 3. Header Configuration

* Header display options:

  * Organization name only
  * Organization logo only
  * **Organization logo and name** (selected)

This controls what users see at the top of the portal.

---

### 4. Upload Branding Assets

* **Logo for theme-color background**
* **Logo for white/light background**
* **Brand image**

These images ensure consistent branding across light and dark UI modes.

---

### 5. Support Information

Add IT support details visible to users:

* Contact name
* Phone number
* Email address
* Website name
* Website URL
* Additional support information

This helps users reach IT without external documentation.

---

### 6. Privacy Configuration

* **Privacy statement URL**
* Optional **custom privacy message**

Ensures transparency and compliance.

---

### 7. App Sources

Choose which applications are shown:

* Azure AD Enterprise applications
* Office Online applications

---

### 8. Feature Visibility

* Select which features to:

  * Show
  * Hide
* Configure based on organizational requirements

---

### 9. Review and Save

1. Click **Review + Save**
2. Verify configuration summary
3. Click **Save**

---

## Confirmation

* Message displayed:

  * *Go to Default customization policy to view the details*
* Branding policy is now active

---

Branding appears consistently across **web and app experiences**.
